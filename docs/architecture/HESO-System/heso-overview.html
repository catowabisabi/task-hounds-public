The Big Idea

One Mind, One Worker

A worker that reviews its own work always says "looks fine." HESO splits the two jobs in two: Hermes does all the thinking, judging and quality control; Sisyphus only executes. The separation is the whole trick.

The Mind
Hermes — messenger of the gods

Hermes

  • Project Manager — sets priorities & the TODO board
  • Product Owner — guards user intention, proposes ideas
  • QA / Reviewer — reads the actual code, verifies every fix
  • Commits & pushes itself (most reliable)
  • Sends you one short report each cycle
⟶
The Hands
Sisyphus — endless, tireless labour

Sisyphus

  • Worker — executes only what Hermes dispatches
  • Fixes the code, reports back a summary
  • Never reviews itself, never sets priorities
  • Spins up sub-agents to finish faster
The Loop

Observe → Fix → Repeat, Forever

Every cron cycle, Hermes wakes, looks at reality, decides what matters, hands Sisyphus a single clear task, then verifies and reports. Like the myth — the work never ends, but here it compounds.

Why It's Different

Not Another Linter

Self-healing

It doesn't just report problems — it fixes them, commits, and moves on. No human in the inner loop.

Never sleeps

800–1000 cycles a day. Progress compounds while you do anything else.

Generates its own ideas

Built-in brainstorming means it proposes features, safety and performance work — not just bug fixes.

Featherweight

No vector DB, no embeddings, no GPU tax. Plain SQLite + tmux + cron. Runs anywhere.

Where It's Going

Roadmap

HESO is open source and built in the open. Here's the direction.

Near-term · v1.x

Solid & visible

  • Risk / approval gates — high-risk changes (schema, auth) need a one-tap Telegram approval before dispatch.
  • Web dashboard — one screen for all jobs, the TODO board and commit history.
  • Cost & GPU telemetry — track resource use per cycle to keep loops cheap.
Mid-term · v2

Scale & adapt

  • Multi-worker — one Hermes directing many Sisyphus workers across projects and modules.
  • Pluggable workers — swap in Claude Code, Aider or any agent behind the same interface.
  • Self-evolving prompts — Hermes tunes how it dispatches based on what gets the best results.
Long-term

Cross-pollinate

  • Cross-project idea transfer — a concept sparked in one project can inspire another.
  • Optional vector recall — opt-in semantic memory for those with GPU headroom (off by default).
