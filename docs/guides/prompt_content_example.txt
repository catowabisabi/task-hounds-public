你係一個資深 full-stack developer。請幫我建立一個簡單但實用嘅「新聞 Web App」。

## 目標

我要一個可以每日睇新聞、分類新聞、搜尋新聞、儲存重要新聞嘅 web app。整體要 simple、乾淨、快、容易維護，唔需要太複雜。

## 技術要求

請使用以下技術：

- Frontend：Next.js
- Backend：Next.js API routes 或 server actions
- Language：TypeScript
- Styling：Tailwind CSS
- Database：SQLite 或 PostgreSQL
- ORM：Prisma
- Package manager：pnpm
- UI：簡潔 dashboard style
- 支援 desktop 同 mobile responsive

## 主要功能

### 1. 新聞列表

首頁顯示新聞列表，每條新聞要有：

- 標題
- 來源
- 發佈時間
- 分類
- 簡短摘要
- 原文連結
- 重要程度分數
- 是否已讀
- 是否收藏

新聞列表要可以按時間由新到舊排序。

### 2. 新聞分類

新聞需要支援以下分類：

- World
- Business
- Technology
- AI
- Crypto
- Stock Market
- Design
- Local News
- Other

用戶可以按分類 filter。

### 3. 搜尋功能

要有 search bar，可以搜尋：

- 標題
- 摘要
- 來源
- 分類

### 4. 新聞詳情頁

每一條新聞可以打開詳情頁，顯示：

- 完整標題
- 來源
- 發佈時間
- 分類
- 摘要
- AI 分析
- 重要關鍵字
- 原文連結
- 收藏按鈕
- 標記已讀按鈕

### 5. AI 分析功能

每條新聞需要有 AI 分析欄位，格式如下：

- 一句話總結
- 為什麼重要
- 可能影響
- 相關行業
- 重要關鍵字
- 重要程度分數：1-10

AI 分析要用 function 寫好，方便之後接 OpenAI API / local LLM。

暫時可以先用 mock function，不需要真 API key，但要保留清楚接口。

### 6. 新聞新增功能

要有一個簡單 admin page，可以手動新增新聞。

欄位包括：

- title
- source
- url
- publishedAt
- category
- summary
- content
- keywords

提交後寫入 database。

### 7. 收藏功能

用戶可以收藏新聞。

收藏後可以喺 `/saved` 頁面睇返所有收藏新聞。

### 8. 已讀功能

用戶可以將新聞標記為已讀 / 未讀。

列表上要清楚顯示狀態。

### 9. Dashboard 統計

首頁上方要有簡單統計卡：

- 今日新聞數量
- 未讀新聞數量
- 收藏新聞數量
- 高重要性新聞數量，例如 score >= 8

### 10. UI 要求

UI 風格：

- simple
- clean
- dashboard style
- 類似 Notion / Linear / Vercel dashboard
- 白底或深色模式都可以
- card layout
- mobile friendly
- 不要太多動畫
- 不要 over-design

每個新聞 card 要容易 scan。

## 頁面結構

請建立以下 routes：

- `/`  
  新聞 dashboard

- `/news/[id]`  
  新聞詳情頁

- `/saved`  
  收藏新聞

- `/admin/new`  
  手動新增新聞

## API / Server Actions

請建立以下功能：

- get all news
- get news by id
- create news
- update read status
- update saved status
- search news
- filter by category
- get dashboard stats
